

# bank OCR
Congratulations ! You now work in a bank.

Good news, the bank just purshased a new machine that scans account number from cheques. It will save lots of manual tasks.

Bad news, the format generated by the machine is not compatible with our systems. One other thing, the machine is not 100% accurate and sometimes makes mistakes...

Your job is to write a program that will read the file from the scanner, translate it, check for error and write a report.

### Input format
        _  _     _  _  _  _  _
      | _| _||_||_ |_   ||_||_|
      ||_  _|  | _||_|  ||_| _| 


Each entry is 4 lines long, and each line has 27 characters. The first 3 lines of each entry contain an account number written using pipes and underscores, and the fourth line is blank. Each account number should have 9 digits, all of which should be in the range 0-9. A normal file contains multiples entries.

### Checksum

To be able to check if a scanned account number is correct, you can run a checksum as follow

    account number:  3  4  5  8  8  2  8  6  5
    position names:  d9 d8 d7 d6 d5 d4 d3 d2 d1

    checksum calculation:
    (1*d1 + 2*d2 + 3*d3 + 4*d4 + 5*d5 + 6*d6 + 7*d7 + 8*d8 + 9*d9) mod 11 = 0

You should be very careful to read the definition of checksum correctly. The digits are reversed from what you expect.

### Output form

The bank system is expected to read a file with this format : 

    457508000
    664371495 ERR
    86110??36 ILL

ERR --> checksum is in error
ILL --> could not read account number
